What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. The DOM represents the document as nodes and objects; that way, programming languages can interact with the page.

A web page is a document that can be either displayed in the browser window or as the HTML source. In both cases, it is the same document but the Document Object Model (DOM) representation allows it to be manipulated. As an object-oriented representation of the web page, it can be modified with a scripting language such as JavaScript.

How do you select an element from the DOM?

There are various differnt types of DOM methods are exist. The most common method is document.getElementById() method. We can easily use the Document interface returns an Element object representing the element whose id property matches the specified string. Since element IDs are required to be unique if specified, they're a useful way to get access to a specific element quickly.

Another way of selecting an element is by using the document.querySelector() method and passing a CSS selector as an argument.

The document.querySelectorAll() method works in a similar way to .querySelector(), but it returns an array-like object of all the elements that match the CSS selector.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in the DOM where a single event listener is attached to a parent element to manage events triggered by its child elements. Instead of attaching individual event listeners to each child element, the parent listens for events that "bubble up" from the children, making the parent responsible for handling those events

How it works?

  1. Event Bubbling: When an event occurs on a child element, it first runs its own event handler, then it "bubbles up" to its parent, and eventually to the root of the document (document), triggering event handlers along the way.

  2. Event Delegation: By attaching an event listener to the parent element, you can capture events triggered by its child elements. Using the event.target property, you can determine which child element triggered the event and respond accordingly.
